Phase 2 — Core Features Built

YouTube Downloading

    Integrated yt-dlp for downloading YouTube audio and full video
    Route: artifacts/api-server/src/routes/youtube.ts
    Supports audio-only (for slideshow mode) and full video (for smart overlay mode)

AI Timeline Analysis (Gemini)

    Integrated Google Gemini AI to analyse YouTube audio/transcripts
    Gemini segments the video into time ranges with image prompts, bhajan detection, and image change frequency
    Returns a structured timeline: [{ startSec, endSec, isBhajan, imageChangeEvery, imagePrompt, description }]
    Timeline is sorted, gaps filled, overlaps resolved before use

AI Image Generation

    For each timeline segment, Gemini generates photorealistic devotional images
    Multiple images per segment supported (based on imageChangeEvery)
    Images generated in parallel alongside audio downloading to save time

FFmpeg Video Rendering

    Three render paths depending on mode:
        Single image — loop image + audio with fade in/out
        Slideshow (multiple images) — concat demuxer + chained fade filters
        Video overlay (smart + YouTube URL) — AI images overlaid on real video footage using filter_complex + overlay with time-gated enable expressions
    Audio trimming for clip-specific rendering (re-encoded to AAC to avoid format conflicts)
    Gap filling: short gaps between clips padded with previous image or black frame
    Tail padding: if images run short of audio duration, last image or black frame extended to cover

Password Protection

    All Bhagwat routes protected with a password (bhagwatnarrationvideos@clips2026, overridable via BHAGWAT_PASSWORD secret)

Progress Tracking

    Server-Sent Events stream reports live percent + message to the frontend throughout: download → image gen → render → done

Phase 4 — Black Video Bug (Root Cause Analysis)

Symptom: Rendered videos were completely black for the entire duration

Root cause identified: FFmpeg's fade=t=out filter has a permanent zero-brightness side effect — after it finishes fading to black, it holds all frames at brightness 0 for the rest of the video. When multiple fades are chained in one -vf string (one per clip boundary), the first fade=t=out zeroes the signal, and every subsequent fade=t=in multiplies against zero — stays black. Nothing visible after the first boundary.
Phase 5 — Black Video Bug Fix Attempts

First attempt (wrong): Per-clip rendering

    Each image rendered into its own .mp4 with isolated fades, then all clip videos concatenated
    This fixed the black screen but increased render time by ~5x (multiple FFmpeg passes)
    User rejected this — "I said don't change the render logic"
    Reverted completely

Second attempt (correct): enable expressions on each fade filter

    Kept the original architecture 100%: concat demuxer, single FFmpeg pass, same codec/quality/audio settings
    Added only :enable='...' to each fade filter to confine it to its own time window
        Opening fade-in: enable='lte(t,3.0)'
        Each boundary fade-out: enable='between(t,foStart,boundary)'
        Each boundary fade-in: enable='between(t,boundary,fiEnd)'
        Closing fade-out: enable='gte(t,lastFoStart)'
    When a fade is outside its window, the enable=0 causes FFmpeg to pass frames through untouched — no held-black bleed
    Fix confirmed working, performance identical to original

Phase 6 — Long Video Analysis (Pending Fixes)

Identified 5 potential issues for 7–15 minute videos, no edits made yet:

    Timing drift — Math.round() on AI timestamps causes ±0.5s error per segment; compounds to 5–15s image/audio desync on long videos
    Video overlay OOM — Smart mode with YouTube URL loads all AI images simultaneously for the full duration; 112 images on a 15-min video can crash FFmpeg with out-of-memory
    No mid-video resync — Slideshow relies purely on clip durations with no correction anchor; drift built up mid-video cannot be corrected
    Single-image path has old chained fades — Harmless in practice (video ends after fade-out) but technically same old pattern
    Gap fill tolerance (0.5s) — Cosmetic only; tiny gaps swallowed silently
