Here are the 5 issues in plain language:

1. Images drift out of sync on long videos

The AI gives timestamps like "show this image from 9.7s to 20.3s" — but the code rounds those to whole numbers (10s to 20s). Each clip is off by a tiny bit. On a short video you don't notice. On a 15-minute video with 90 clips, those tiny errors stack up — by the end, the image of "Ram crossing the river" could appear 10–15 seconds after the narrator actually says it. Audio is always correct, only the pictures drift.

2. Smart mode (YouTube URL) can crash or freeze on long videos

When you use Smart mode with a YouTube link, all AI-generated images are loaded into memory at the same time for the full video duration. For a 15-minute video that could be 100+ images all held in RAM simultaneously. FFmpeg runs out of memory, crashes, and you get a "killed by system" error with no useful explanation. Shorter videos are fine; 10–15 minutes is where this breaks.

3. No way to re-sync pictures mid-video

The video is built by placing each image for a set number of seconds in sequence. If a few clips are slightly wrong duration (from issue 1), the error just keeps growing with no correction. There is no checkpoint that says "at 5 minutes, make sure the right image is showing." It's like a train with no stations — if it falls behind, it never catches up.

4. Single-image videos have an old fade pattern (low risk)

When a video has only one image, it still uses the old fade style that was the root of the black screen bug. In practice it's harmless — the video ends right after the fade-out so nothing gets blacked out. But it's the same code pattern that caused the original problem, just lucky in this case.

5. Tiny gaps between images are silently hidden

If two clips have a gap smaller than half a second between them, the code just stretches the previous image to cover it without telling you. Usually invisible. On a long video with lots of rounding (from issue 1), you might see an image hold slightly longer than expected before the next one appears — no error, just a subtle wrongness.

Which ones actually matter for you:

    Issues 1 and 3 together are the main reason a 15-minute video looks "off" — pictures at wrong times
    Issue 2 is the one that causes hard crashes on long videos
    Issues 4 and 5 are minor and unlikely to cause visible problems
